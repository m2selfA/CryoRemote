Tool: CryoRemote

CryoRemote provides an SSH/SFTP-oriented RELION-first project browser inside ChimeraX. It imports direct OpenSSH aliases from ~/.ssh/config, browses remote files, indexes RELION projects from remote default_pipeline.star, previews metadata and MRC headers, and caches maps/models locally before opening them in ChimeraX.

Phase 1 capabilities

  • Connect to direct SSH aliases with public keys, agent support, password fallback, and keyboard-interactive prompts
  • Browse remote directories with lazy loading
  • Show RELION projects as both a pipeline flowchart and a job table
  • Refresh RELION status inside the active ChimeraX session with a session-bound watcher
  • Preview .star, .txt, .log, .json, .out, .err, and .cxc remotely with execution confirmation
  • Preview MRC/MAP headers without downloading the full file
  • Cache .mrc, .map, .pdb, .cif, and source copies of remote .cxc files locally before opening
  • Open a remote .cxc by rewriting supported open ... file operands to local cache paths and then opening the rewritten local command file in ChimeraX
  • Recognize common RELION layouts and expose actions for latest refine maps, last completed jobs, half maps, and postprocess+model pairs
  • Show related files, parent/child jobs, and next-action hints for the selected RELION job
  • Use a bundled Agnes-generated Scientific Cryo Blue asset pack for branding, empty-state illustration, and action icons

Limitations

  • ProxyJump, ProxyCommand, Include, and Match are detected and warned about, but not implemented in phase 1
  • Streaming full maps is not implemented; maps are cached locally before opening
  • Remote .cxc support only rewrites the leading file operands of open ...; nested .cxc, forEachFile, coords, and other option-level path semantics are not rewritten
  • Detached background watchers, cryoSPARC .cs, Slurm integration, and arbitrary remote command execution are deferred
